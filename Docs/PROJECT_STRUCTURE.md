# Project Structure Documentation

This document describes the main driver and implementation files for each part of the project.

## Part 1: Map

- **Driver File:** `MapDriver.cpp`
- **Utilized Files:** `Models/Map.h`, `Models/Map.cpp`
- **Description:**
  - `MapDriver.cpp` serves as the entry point for testing and demonstrating the map functionality. It relies on the map class definitions and implementations provided in `Map.h` and `Map.cpp`.

## Part 2: Player

- **Driver File:** `PlayerDriver.cpp`
- **Utilized Files:** `Models/Player.h`, `Models/Player.cpp`
- **Description:**
  - `PlayerDriver.cpp` is used to test and showcase the player-related features, utilizing the player class and its implementation.

## Part 3: Order List

- **Main File:** `Models/Orders.cpp`
- **Utilized Files:** `Models/Orders.h`
- **Description:**
  - The order list functionality is implemented in `Orders.cpp` and defined in `Orders.h`.

## Part 4: Card Deck/Hand

- **Driver File:** `CardsDriver.cpp`
- **Utilized Files:** `Models/Cards.cpp`, `Models/Cards.h`
- **Description:**
  - `CardsDriver.cpp` demonstrates the card deck and hand features, using the card class and its implementation.

## Part 5: Game Engine

- **Driver File:** `GameEngineDriver.cpp`
- **Utilized Files:** `Models/GameEngine.cpp`, `Models/GameEngine.h`
- **Description:**
  - `GameEngineDriver.cpp` is responsible for running and testing the game engine logic, utilizing the game engine class and its implementation.
